Detailed CSS Tutorial

Understanding the Functionality of HTML: A deep dive into HTML structure, components, and best practices.

1. Understanding the Functionality of CSS

1.1 Purpose of CSS

CSS stands for Cascading Style Sheets and is the language used to describe the presentation of a document written in HTML or XML. Its primary function is to separate content from design. This means that developers write simple HTML to describe the content and use CSS to dictate how that content should appear on the page.

The key purposes of CSS include:

  • Formatting and Layout:

    CSS provides a wide range of properties that allow you to control the structure and visual presentation of your content. This includes setting fonts, colors, spacing, and positioning of elements. Modern layout techniques such as Flexbox and CSS Grid let you design complex, responsive layouts without relying on table-based designs.

  • Separation of Concerns:

    Instead of merging content with presentation, CSS allows you to keep your HTML clean and semantic by removing style-related code. This separation makes both the HTML and CSS easier to read, maintain, and update. It also encourages reusability, as the same stylesheet can be applied across multiple pages.

  • Enhanced Maintainability:

    Because styling is centralized in one or more CSS files, making global changes becomes much easier. Instead of updating inline styles in several different HTML files, you can change the stylesheet to instantly update the presentation across your entire website. This efficiency is especially beneficial in large projects.

1.2 Example: Styling Text and Layout

Below is an example demonstrating how CSS can be used to style text elements and create a basic layout using a flexible container and boxes. This example shows how you can set fonts, colors, and layout properties to achieve a clean and modern design. 

  
  /* Example CSS for Styling Text and Layout */
  
  body {
    font-family: 'Arial', sans-serif;
    background-color: #f4f4f4;
    margin: 0;
    padding: 0;
  }
  
  h1 {
    color: #333;
    text-align: center;
    margin-top: 20px;
  }
  
  .container {
    width: 80%;
    margin: 20px auto;
    display: flex;
    flex-wrap: wrap;
    justify-content: space-around;
  }
  
  .box {
    background-color: #fff;
    border: 1px solid #ccc;
    padding: 10px;
    width: 30%;
    margin-bottom: 20px;
  }

In the above CSS:
body sets a global font, background color, and resets default margins.
h1 centers the heading and applies a dark color.
.container uses Flexbox to arrange child elements responsively.
.box styles individual content boxes with padding, a border, and a defined width.

1.3 Referencing CSS in HTML

There are three main methods to reference CSS in an HTML document. Each method has its own use cases and advantages:

  1. Inline Styles

    Inline styles are written directly into an HTML element’s style attribute. This method is useful for quick, one-off changes. However, it is not recommended for extensive styling because it mixes content with presentation and reduces maintainability.

  2. Internal Style Sheets

    An internal style sheet is defined within a <style> tag in the <head> section of your HTML document. This approach is ideal for styling a single HTML document or when you want to include styles that are unique to that page.

    1.4 CSS Syntax

    CSS follows a straightforward syntax where each rule-set consists of a selector and a declaration block:

    mc
    • Selectors:

      Selectors tell the browser which HTML elements the styles should be applied to. They can be as simple as targeting a single element (e.g., p for paragraphs) or more complex (e.g., ul li for list items within a list, .class for elements with a specific class, or #id for a specific element with an id).

    • Declarations:

      Declarations are the instructions that apply styles to the selected elements. Each declaration comprises a CSS property and its corresponding value, such as color: red;. Multiple declarations can be grouped within a declaration block enclosed by curly braces.

    Consider the following example that illustrates selectors and declarations: 

      
  /* Examples of basic CSS syntax */
  
  /* Element Selector */
  p {
    color: red;
    font-size: 16px;
  }
  
  /* Class Selector */
  .highlight {
    background-color: yellow;
  }
  
  /* ID Selector */
  #unique {
    margin-top: 20px;
  }
  
  /* Combined Selector: Targeting only paragraphs with the "highlight" class */
  p.highlight {
    font-weight: bold;
  }

    In this example, the p selector targets all paragraphs, while .highlight and #unique target elements by their class and id respectively. The last rule, p.highlight, applies styles only to those paragraphs that have the class "highlight."

2. CSS Selectors

• Basic Selectors

o Type Selectors

Type selectors target elements by their tag name. For example, using p targets all paragraph elements. These selectors are the most basic and are extremely useful for setting global styles to elements defined in HTML.

Example:


  /* This rule applies to all paragraph elements */
  p {
    font-size: 16px;
    line-height: 1.5;
    color: #333;
  }

In this snippet, every <p> element in the document is styled consistently. No matter where the paragraphs appear in your HTML, they will inherit these properties. This is especially useful when you have a uniform design requirement.

o Class Selectors

Class selectors are used to target elements with a specific class attribute. They begin with a dot (.) followed by the class name. This method allows you to apply particular styles to groups of elements even if they are of different types.

Example:


  /* This rule applies to elements with class "highlight" */
  .highlight {
    background-color: #fffbcc;
    padding: 10px;
    border-radius: 4px;
  }

Here, any element with the class highlight will receive the specified background, padding, and border radius styling. It is a powerful tool for emphasizing content.

o ID Selectors

ID selectors target a single unique element by its id attribute. They start with a hash (#) followed by the id name. Unlike classes, IDs are intended to be unique within the HTML document.

Example:


  /* This rule applies to the element with the id "main-header" */
  #main-header {
    font-size: 2em;
    color: #005b96;
    margin-bottom: 20px;
  }

Because IDs are unique, they have a higher specificity than type or class selectors, meaning their rules will generally override others. Use ID selectors when you need to target a single, unique element on your page.

o Universal Selector

The universal selector is represented by an asterisk (*) and selects every element in the document. It is incredibly powerful, but must be used with caution to avoid performance issues and unintended styling.

Example:


  /* Resets margin and padding for every element */
  * {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
  }

This example is common in CSS resets, ensuring that every element starts without default browser styles, which helps create a consistent baseline across different browsers.

o Attribute Selectors

Attribute selectors target elements based on the presence or value of an attribute. This can be used for advanced selection when you need to style elements with a specified attribute or attribute value.

Example:


  /* Style all input elements with type="text" */
  input[type="text"] {
    border: 1px solid #ccc;
    padding: 8px;
    border-radius: 4px;
  }
  
  /* Style any element with a title attribute */
  [title] {
    text-decoration: underline;
  }

The first example specifically targets text inputs, while the second applies to any element that contains the title attribute. Attribute selectors provide an additional way to precisely target elements beyond their tag, class, or id.

• Combining Selectors

Combining selectors allows you to apply multiple conditions to target elements more precisely. This enhances the reusability and efficiency of your style sheet.

o Combinator Types

Child Combinator

The child combinator is represented by the > symbol and is used to target direct children of a specified element. This means that only elements that are immediate children of the parent will be selected.

Example:


  /* Select only direct 
  •  children of 
       */
  ul > li {
    color: #007acc;
    padding: 5px 0;
  }

    • In this example, only list items that are a direct child of a <ul> are affected. Deeper nested <li> elements will not inherit this style.

    Descendant Combinator

    The descendant combinator is a single space. It selects all elements that are descendants (at any level) of a specified element.

    Example:

    
  /* Select all  elements within any 
     */
  div span {
    background-color: #e0f7fa;
    padding: 2px 4px;
  }

    Unlike the child combinator, this rule will apply to <span> elements nested at any level inside a <div>.

    Adjacent Sibling Combinator

    The adjacent sibling combinator is represented by the plus sign (+). It selects an element that is directly adjacent to another element.

    Example:

    
  /* Select the paragraph that immediately follows an 
     */
  h3 + p {
    font-style: italic;
    margin-top: 0;
  }

    In this example, only the paragraph immediately following an <h3> will be styled in italic.

    General Sibling Combinator

    The general sibling combinator is represented by the tilde (~) and selects all siblings that follow the specified element, regardless of whether they are immediately adjacent.

    Example:

    
  /* Select all 
     elements that are siblings of an 
     */
  h2 ~ p {
    color: #555;
    line-height: 1.8;
  }

    In this example, every <p> that follows an <h2> (and shares the same parent) will receive these styles.

    • Writing Efficient Style Sheets

    o Cascading

    Cascading refers to how the browser combines multiple CSS rules that apply to the same element. Rules are applied based on their order, source, and specificity. The cascading nature ensures that if multiple styles conflict, a clear hierarchy determines which style takes precedence.

    Example Explanation:

    
  /* Global style for paragraphs */
  p {
    color: black;
  }
  
  /* More specific rule */
  div p {
    color: darkblue;
  }

    In the above example, if a <p> element lies within a <div>, it will be styled with darkblue even though a general rule sets all paragraphs to black. This is a direct result of the cascade and more specific selector.

    o Specificity

    Specificity determines which CSS rule is applied when multiple rules could match the same element. It is calculated based on the types of selectors. In simple terms, IDs have the highest specificity, followed by classes and attributes, and finally type selectors.

    Example:

    
  /* Type selector - specificity of 1 */
  p {
    color: black;
  }
  
  /* Class selector - specificity of 10 */
  .highlight {
    color: green;
  }
  
  /* ID selector - specificity of 100 */
  #special {
    color: red;
  }

    In the above example, an element with id="special" and class highlight will be red because the ID selector has the highest specificity.

    • Colors in CSS

    o Named Colors

    Named colors are the basic color names (like red, blue, and green) that browsers support by default. They are convenient for quick styling but offer less precision compared to other color formats.

    Example:

    
  /* Using named colors for styling */
  h1 {
    color: navy;
  }
  p {
    color: gray;
  }

    In the example, navy and gray are named colors that will render a blue and gray hue respectively.

    o Hexadecimal Codes

    Hexadecimal color codes provide a way to specify colors using a six-digit notation preceded by a hash (#). Each pair of digits represents the intensity of red, green, and blue components respectively (ranging from 00 to FF).

    Example:

    
  /* Example using hex colors */
  div {
    background-color: #ff5733; /* A vibrant orange-red */
  }

    Here, #ff5733 is a hex code where FF represents red at maximum intensity, and 57 and 33 define the amounts of green and blue.

    o RGB and HSL Values

    RGB (Red, Green, Blue) values define colors by specifying the intensity of each component on a scale from 0 to 255. HSL (Hue, Saturation, Lightness) values offer an alternative in which hue is an angle (0 to 360 degrees), saturation and lightness are percentages.

    Example using RGB:

    
  /* Styling using RGB */
  header {
    background-color: rgb(10, 84, 255);
  }

    Example using HSL:

    
  /* Styling using HSL */
  header {
    background-color: hsl(220, 100%, 50%);
  }

    Both examples above result in a similar cool blue background, giving you multiple ways to achieve the same visual outcome.

    • Gradients in CSS

    Gradients allow you to create smooth transitions between two or more specified colors. They can be used as backgrounds for elements or as overlays to achieve visually appealing designs.

    o Linear Gradients

    Linear gradients create a transition along a straight line. You can specify the direction (to right, to bottom, etc.) or use angles.

    Example:

    
  /* Linear gradient background */
  .button {
    background: linear-gradient(to right, #0A84FF, #0057e7);
    border: none;
    color: #fff;
    padding: 10px 20px;
    border-radius: 4px;
    cursor: pointer;
  }

    In this example, the button background smoothly transitions from a bright blue (#0A84FF) to a deeper blue (#0057e7) from left to right.

    o Radial Gradients

    Radial gradients emanate from a central point and radiate outward in a circular or elliptical shape.

    Example:

    
  /* Radial gradient background */
  .card {
    background: radial-gradient(circle, #ffffff, #d3d3d3);
    padding: 20px;
    border-radius: 8px;
    box-shadow: 0 4px 10px rgba(0, 0, 0, 0.1);
  }

    This creates a subtle light-to-dark gradient effect, ideal for card or container backgrounds.

    o Conic Gradients

    Conic gradients (or angular gradients) create a gradient with a circular progression through hues around a center point.

    Example:

    
  /* Conic gradient used as a circular background */
  .circle {
    width: 150px;
    height: 150px;
    border-radius: 50%;
    background: conic-gradient(from 0deg, #0A84FF, #0057e7, #0A84FF);
  }

    In this example, the gradient changes around the circumference of the circle, creating a dynamic and colorful effect.

    3. CSS Styling Essentials

    • Styling Text

    o Color, Alignment, Font

    Text styling in CSS is fundamental for creating readable and visually engaging content. The following properties are often used:

    • color: Defines the color of the text. Accepts named colors, hexadecimal, RGB, or HSL values.
    • text-align: Aligns the text horizontally within its container (e.g., left, right, center, or justify).
    • font-family: Specifies the typeface for the text, with options to fall back to generic families.

    Example Code:

    
  /* Applying color, alignment, and font style to text */
  p {
    color: #007acc;           /* Sets text color to blue */
    text-align: justify;       /* Justifies the text for neat alignment */
    font-family: 'Arial', sans-serif; /* Uses Arial with sans-serif fallback */
  }

    In this snippet, every paragraph is styled with a specified blue color, justified alignment, and Arial typeface to ensure consistency and readability.

    o Line Height, Font Size, Font Family

    Beyond color and alignment, controlling the space between lines (line-height) and the size of text (font-size) is essential. Adjusting the font family further refines typography.

    Example Code:

    
  /* Adjusting line height and font size for improved readability */
  p {
    line-height: 1.8;                  /* Increases space between lines */
    font-size: 18px;                   /* Sets a comfortable reading size */
    font-family: 'Georgia', serif;     /* Uses a classic serif font for elegance */
  }

    These properties work together to make the text comfortable to read while harmonizing with the overall design.

    • Styling Links

    Links are interactive elements that require distinct styling based on their state. CSS offers pseudo-classes that enable you to style links in different states:

    o Unvisited, Visited, Hover, Active States

    The primary pseudo-classes for links are:

    • :link - Targets unvisited links.
    • :visited - Targets links that the user has already visited.
    • :hover - Applies when the user hovers the mouse pointer over the link.
    • :active - Applies during the moment the link is being clicked.

    Example Code:

    
  /* Styling various states of a link */
  a:link {
    color: #0A84FF;         /* Blue for unvisited links */
    text-decoration: none;
  }
  
  a:visited {
    color: #005bb5;         /* Darker blue for visited links */
  }
  
  a:hover {
    text-decoration: underline; /* Underlines on hover for clarity */
    color: #003f7f;
  }
  
  a:active {
    color: #ff4500;         /* A contrasting color during activation */
  }

    This example demonstrates how careful link styling can provide clear interaction feedback and maintain a cohesive aesthetic throughout the website.

    • Styling Lists

    Lists structure information using ordered (<ol>) or unordered (<ul>) elements. Default styling can be adjusted to enhance visual appeal.

    o Unordered and Ordered Lists

    Unordered lists typically display with bullet points, whereas ordered lists show numbers or letters. You can customize list markers, padding, and indentation:

    Example Code:

    
  /* Standard list styles */
  ul {
    list-style-type: disc; /* Bullet points */
    padding-left: 20px;
  }
  
  ol {
    list-style-type: decimal; /* Numbers */
    padding-left: 20px;
  }
  
  /* Custom list style for a fancy look */
  ul.custom {
    list-style: none; /* Remove default markers */
  }
  
  ul.custom li::before {
    content: "•";
    color: #0A84FF;
    margin-right: 10px;
  }

    These examples illustrate how to modify the default look of lists to match your site's design language.

    • Styling Tables

    Tables are used to display structured data. Styling tables involves managing borders, padding, background colors, and alternate row colors to improve data readability.

    o Basic Table Styles

    Common practices include setting border-collapse to collapse borders, applying uniform padding to cells, and adding background colors to header cells or alternating rows.

    Example Code:

    
  /* Basic table styling */
  table {
    border-collapse: collapse;
    width: 100%;
  }
  
  th, td {
    border: 1px solid #ddd;
    padding: 8px;
  }
  
  th {
    background-color: #f2f2f2;
    text-align: left;
  }
  
  /* Apply alternate row colors for better readability */
  tr:nth-child(even) {
    background-color: #f9f9f9;
  }

    The above code creates a clean and structured table layout that is both accessible and visually appealing.

    • Styling Images

    CSS not only controls text and layout but also how images appear. You can manage borders, sizes, and positioning easily.

    o Borders, Sizing, Positioning

    Key properties for image styling include:

    • border: Adds a border around the image. You can control width, style, and color.
    • width/height: Control the size of the image. Using auto for height maintains aspect ratio.
    • float: Positions the image to one side so that text can wrap around it.

    Example Code:

    
  /* Styling an image for a polished look */
  img {
    border: 2px solid #ddd;
    border-radius: 4px;
    width: 300px;        /* Fixed width for consistency */
    height: auto;        /* Maintain aspect ratio */
    float: left;         /* Float image to the left */
    margin-right: 20px;  /* Spacing between image and text */
  }

    This example demonstrates adding subtle borders, controlling the image size, and using float positioning for an integrated layout.

    • Positioning Elements

    CSS positioning determines how elements are placed within the layout. There are several positioning types to choose from:

    o Static, Relative, Absolute, Fixed

    - Static: The default positioning method, where elements flow naturally with the document.
    - Relative: Positions an element relative to its normal position, enabling slight offsets.
    - Absolute: Removes the element from normal flow and positions it relative to its nearest positioned ancestor.
    - Fixed: Positions the element relative to the viewport so it stays in place during scrolling.

    Example Code:

    
  /* Demonstrating various positioning types */
  
  /* Static (default), no positioning applied */
  .static-box {
    position: static;
    background-color: #f0f0f0;
    padding: 10px;
  }
  
  /* Relative positioning for slight adjustment */
  .relative-box {
    position: relative;
    top: 10px;
    left: 20px;
    background-color: #d0ebff;
    padding: 10px;
  }
  
  /* Absolute positioning for precise control */
  .absolute-box {
    position: absolute;
    top: 100px;
    right: 50px;
    background-color: #ffd1dc;
    padding: 10px;
  }
  
  /* Fixed positioning for elements like sticky navbars */
  .fixed-box {
    position: fixed;
    bottom: 0;
    left: 0;
    width: 100%;
    background-color: #cce5ff;
    padding: 10px;
  }

    Each positioning method is useful for different scenarios: static elements follow normal flow, relative allows for fine-tuning, absolute is ideal for overlays or pop-ups, and fixed keeps elements visible on scroll.

    • Floating and Overlapping Elements

    To create dynamic layouts, CSS allows you to float elements and manage overlapping content using z-index.

    o Float, Z-index

    - Float: Removes an element from the normal flow and allows text or inline elements to wrap around it.
    - Z-index: Establishes the stacking order for positioned elements. Higher values appear on top.

    Example Code:

    
  /* Floating an image for text wrapping */
  .float-image {
    float: left;
    margin-right: 15px;
  }
  
  /* Overlapping boxes using z-index */
  .overlap-container {
    position: relative;
  }
  
  .overlap-box {
    position: absolute;
    top: 20px;
    left: 20px;
    width: 150px;
    height: 150px;
    background-color: rgba(10, 132, 255, 0.8);
    z-index: 10; /* This box appears on top */
  }
  
  .overlap-box-secondary {
    position: absolute;
    top: 50px;
    left: 50px;
    width: 150px;
    height: 150px;
    background-color: rgba(255, 59, 48, 0.8);
    z-index: 5;  /* This box appears below the primary one */
  }

    In this example, if both overlapping boxes share the same container, the one with the higher z-index will be visible on top.

    • The Box Model

    The CSS box model is central to layout design. It defines how the size and spacing of elements are calculated.

    o Content, Padding, Border, Margin

    The box model consists of:

    • Content: The actual content such as text or images.
    • Padding: The space between the content and the border. It ensures that text does not touch the border.
    • Border: A visible boundary that surrounds the padding and content. Its thickness, style, and color can be customized.
    • Margin: The outermost space that separates the element from other elements.

    Example Code:

    
  /* Demonstrating the CSS box model */
  .box-model-example {
    width: 300px;             /* Width of the content area */
    padding: 20px;            /* Space between the content and border */
    border: 5px solid #333;   /* Border surrounding the element */
    margin: 30px;             /* Space outside the element */
    background-color: #f9f9f9;
  }
  
  /*
  Visual Representation:
  [ Margin ]
    [ Border ]
      [ Padding ]
        [ Content ]
  */

    By adjusting each of these properties, you can control the overall layout, spacing, and visual balance of your design. Understanding the box model is critical for building responsive interfaces and ensuring consistent behavior across different browsers.

    4. Advanced CSS Techniques

    • Display Property

    The display property in CSS determines how an element is rendered in the document flow. It affects layout, visibility, and interactivity of elements. Below, we examine three common display types: block, inline, and grid.

    o Block

    A block-level element starts on a new line and takes up the full available width. Common block elements include <div>, <p>, and <h1>–<h6>. They also accept margin and padding.

    Example:

    
  /* All paragraphs are block-level elements by default */
  p {
    display: block;
    background-color: #e0e0e0;
    margin-bottom: 10px;
    padding: 10px;
  }

    In this example, every <p> starts on its own line, has a light background, and consistent spacing.

    o Inline

    Inline elements do not start on a new line; they flow within other elements. Common inline elements include <span>, <a>, and <strong>. Inline elements ignore top and bottom margins or paddings.

    Example:

    
  /* Using inline display for emphasis within a paragraph */
  span.important {
    display: inline;
    color: #d63384;
    font-weight: bold;
  }

    Here, the .important class applied to a <span> alters its color and weight without forcing a new line.

    o Grid

    The grid display enables a two-dimensional layout system with rows and columns. It provides a powerful way to create complex responsive layouts.

    Example:

    
  /* Set an element to grid to enable grid layout */
  .container {
    display: grid;
    grid-template-columns: repeat(3, 1fr); /* Three equal columns */
    grid-gap: 20px;  /* Gap between grid items */
  }

    The above CSS turns the element with class container into a grid, organizing its immediate children into three equally spaced columns.

    • CSS Grid Layouts

    CSS Grid Layout excels at creating complex designs by dividing a container into rows and columns. It enables you to place items precisely within a grid with minimal code.

    o Grid Container and Items

    A grid container is any element with its display property set to grid (or inline-grid). Its direct children become grid items. You can define grid structure via properties like grid-template-columns, grid-template-rows, and grid-gap.

    Example:

    
  /* CSS Grid Example */
  #grid-container {
    display: grid;
    grid-template-columns: repeat(3, 1fr); /* Three equal-width columns */
    grid-template-rows: auto;
    grid-gap: 20px;
    padding: 10px;
    background-color: #f0f0f0;
  }
  
  #grid-container .grid-item {
    background-color: #dcdcdc;
    border: 1px solid #ccc;
    padding: 15px;
    text-align: center;
  }

    In this example, the element with id grid-container holds several grid items organized into three columns with gaps between them.

    • Responsive Design

    Responsive design ensures that your website adapts to various screen sizes and devices. Below are key techniques to achieve responsiveness.

    o Viewport Meta Tag

    The viewport meta tag instructs browsers on how to control the page's dimensions and scaling. It is critical for mobile accessibility.

    Example (placed inside the <head>):

    
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

    This line tells the browser that the width of the page should match the screen's width at a scale of 1.0, ensuring the design works well on all devices.

    o Media Queries

    Media queries allow you to apply CSS rules based on different conditions, such as viewport width, height, pixel density, and more. They are essential for adapting layouts to various devices.

    Example:

    
  /* A media query for devices with a viewport width of 768px or below */
  @media (max-width: 768px) {
    body {
      font-size: 14px;
    }
    #grid-container {
      grid-template-columns: 1fr; /* Single column layout */
    }
  }

    With this media query, the default layout changes when the viewport is smaller than 768 pixels, providing a better user experience on mobile devices.

    o Relative Measurements

    Using relative units like percentages (%), ems, rems, vw (viewport width), and vh (viewport height) creates flexible layouts that adapt dynamically to the screen size.

    Example:

    
  /* A container width set in percentages and font sizes using rem */
  .container {
    width: 80%;         /* 80% of the parent container or viewport */
    padding: 2rem;      /* 2 times the root element's font-size */
    margin: 0 auto;
  }

    This technique ensures your layout scales gracefully on different devices.

    • CSS Animations

    CSS animations enhance the user experience by smoothly transitioning elements from one style to another or applying dynamic transformations. Two key techniques involve transitions and transforms.

    o Transitions

    Transitions enable a change in property values to occur smoothly over a specified duration instead of instantaneously. They can be applied to properties such as color, background, and positioning.

    Example:

    
  /* Smooth background-color transition for a button */
  .button {
    background-color: #0A84FF;
    color: #fff;
    padding: 10px 20px;
    border: none;
    border-radius: 4px;
    transition: background-color 0.3s ease;
  }
  
  .button:hover {
    background-color: #0057e7;
  }

    The transition applied to the .button class creates a smooth color-change effect when the user hovers over the button.

    o Transforms

    Transforms allow you to modify the visual presentation of an element by moving, scaling, rotating, or skewing it. They can be combined with transitions for animated effects.

    Example:

    
  /* Applying a scale and rotation transform on hover */
  .card {
    width: 250px;
